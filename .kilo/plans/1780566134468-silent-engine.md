# Chronicle Bug Fixes Plan

## Issues Summary

### 1. Avatar images not rendering (main page + trace page)
**Root cause:** `useUserAvatar` hook in `app/chronicle/page.tsx` fetches `/api/chronicle/user/[username]` which returns `{ profile: { avatar_url, ... } }`, but the hook only looks for `data.avatar_url || data.user?.avatar_url`. The correct path is `data.profile.avatar_url`.

**Fix:** In `useUserAvatar` (line 63 of `app/chronicle/page.tsx`), update the avatar URL extraction:
```ts
// Before:
const url = data.avatar_url || data.user?.avatar_url || null
// After:
const url = data.profile?.avatar_url || data.avatar_url || data.user?.avatar_url || null
```

### 2. Book covers uploaded to blob storage (no more URL links)
**Root cause:** Both `AddBookPanel` (in `app/chronicle/page.tsx`) and `EditBookPanel` (in `app/chronicle/book/[bookId]/page.tsx`) use a plain text input for `image_url`. We need to replace this with a file input that uploads to Vercel Blob, then stores the resulting URL.

**Approach:**
- Create a new API route `app/api/chronicle/books/cover/route.ts` ‚Äî POST endpoint that accepts `multipart/form-data` with a `cover` file field, uploads to Vercel Blob (under `book-covers/` prefix), and returns `{ url }`.
- In `AddBookPanel`: replace the `image_url` text input with a file picker. On file selection, POST to the new cover upload route and store the returned URL internally. Display a thumbnail preview.
- In `EditBookPanel`: same replacement ‚Äî file picker for upload + show current cover thumbnail if present + allow changing.
- Both panels keep the `image_url` field in the form state (as the blob URL), just the input method changes from text to file.

### 3. Favorites count not working
**Root cause:** `GET /api/chronicle/books` (`app/api/chronicle/books/route.ts`) does not include:
- `favorite_count` ‚Äî no join to `chronicle_favorites`
- `last_traced_at` ‚Äî needed for "Sort by last traced"
- `is_favorited_by_me` ‚Äî needed to show correct heart state on initial load (currently computed client-side from `favoriteIds` state, but `favoriteIds` is loaded separately and might race)

**Fix in `app/api/chronicle/books/route.ts`:** Add LEFT JOINs for:
1. `favorite_count`: count of `chronicle_favorites` rows per book
2. `last_traced_at`: MAX of `chronicle_reviews.created_at` per book
3. `is_favorited_by_me`: boolean based on current session user in `chronicle_favorites`

SQL change:
```sql
SELECT
  b.id, b.title, b.author, b.image_url,
  COALESCE(b.categories, ARRAY[b.category]) AS categories,
  b.added_by, b.added_at,
  COUNT(DISTINCT r.id)::int AS review_count,
  ROUND(AVG(r.rating)::numeric, 1) AS avg_rating,
  COALESCE(json_agg(DISTINCT r.reviewer_name) FILTER (WHERE r.reviewer_name IS NOT NULL), '[]') AS readers,
  COUNT(DISTINCT f.id)::int AS favorite_count,
  MAX(r.created_at) AS last_traced_at,
  BOOL_OR(f.user_name = ${session.user?.name}) AS is_favorited_by_me,
  COALESCE(json_agg(DISTINCT cr.username) FILTER (WHERE cr.username IS NOT NULL), '[]') AS currently_reading
FROM chronicle_books b
LEFT JOIN chronicle_reviews r ON r.book_id = b.id
LEFT JOIN chronicle_favorites f ON f.book_id = b.id
LEFT JOIN chronicle_currently_reading ccr ON ccr.book_id = b.id
LEFT JOIN users cr ON cr.username = ccr.user_name
GROUP BY b.id
ORDER BY b.added_at DESC
```

> **Note:** Currently the `currently_reading` field in the books API response is also missing ‚Äî this also needs to be added (since `BookCard` and the "Currently Reading" banner depend on it).

The client-side `handleFavoriteToggle` already does correct optimistic update on `favorite_count` and `is_favorited_by_me`. The issue is the initial data doesn't have these fields.

### 4. Trace cards missing thumbs up, replies, language flag, Dimensional echoes
**Root cause:** `GET /api/chronicle/books/[bookId]` (`app/api/chronicle/books/[bookId]/route.ts`) fetches reviews with only: `id, reviewer_name, date_read, pages, rating, recommendation, created_at`. Missing:
- `language_read`
- `dimensions` (from a `dimensions` JSON column, or separate table)
- `reactions` (from `chronicle_reactions` table)
- `replies` (from `chronicle_replies` table)

Also, `POST /api/chronicle/reviews` and `PUT /api/chronicle/reviews/[reviewId]` don't save/update `language_read` or `dimensions`.

**Fixes:**

**`app/api/chronicle/books/[bookId]/route.ts` ‚Äî GET:**
Update the review SELECT to include `language_read` and `dimensions`. Then for each review, attach its reactions and replies via additional queries (or subqueries / CTEs):
```sql
SELECT id, reviewer_name, date_read, rating, recommendation, created_at, language_read, dimensions
FROM chronicle_reviews
WHERE book_id = ${bookId}
ORDER BY created_at DESC
```
Then fetch reactions and replies for the book's reviews in bulk:
```sql
SELECT * FROM chronicle_reactions WHERE trace_id = ANY(${reviewIds})
SELECT * FROM chronicle_replies WHERE trace_id = ANY(${reviewIds}) ORDER BY created_at ASC
```
Merge them into each review object before returning.

**`app/api/chronicle/reviews/route.ts` ‚Äî POST:**
Add `language_read` and `dimensions` to the INSERT:
```sql
INSERT INTO chronicle_reviews (book_id, reviewer_name, date_read, rating, recommendation, language_read, dimensions)
VALUES (${book_id}, ${reviewer_name}, ${formattedDate}, ${rating}, ${recommendation}, ${language_read}, ${dimensions})
```

**`app/api/chronicle/reviews/[reviewId]/route.ts` ‚Äî PUT:**
Add `language_read` and `dimensions` to the UPDATE:
```sql
UPDATE chronicle_reviews
SET date_read = ..., rating = ..., recommendation = ..., language_read = ${language_read}, dimensions = ${dimensions}
WHERE id = ${reviewId}
```

- The `next.config.js` has no `images.domains` configured ‚Äî the `<img>` tags using `src` directly (not Next.js `<Image>`) so no next.config change needed for blob URLs.
