# English Translation System Implementation Plan

## Scope
Replace all French user-facing strings with English translations across the entire Papope application. English becomes the default (and currently only) language. Translation infrastructure is put in place so French (or other languages) can be added later.

---

## Phase 1: i18n Infrastructure

### 1.1 Create translation system
- **File**: `lib/i18n/translations.ts`
  - Export a `translations` object: `{ en: { ... }, fr: { ... } }`
  - Organize by namespace: `common`, `keeppushing`, `chronicle`, `games`, `oracle`, `admin`, `ionickel`, `login`
  - Include ALL French strings from the audit with their English equivalents
  
- **File**: `lib/i18n/index.ts`
  - Export `translations` object
  - Export `Language` type (`'en' | 'fr'`)
  - Export `defaultLanguage = 'en'`
  - Export `getTranslation(key, language)` helper for nested key access (e.g., `'keeppushing.nutrition.addSupplement'`)

### 1.2 Create React context + hook
- **File**: `lib/i18n/context.tsx`
  - `I18nProvider` component wrapping `children`
  - `useI18n()` hook returning `{ t, language, setLanguage }`
  - Read initial language from `localStorage` key `papope_lang`, default to `'en'`
  - Persist language changes to `localStorage`

### 1.3 Wrap app in provider
- **File**: `app/providers.tsx`
  - Import and wrap `<I18nProvider>` around existing providers (SessionProvider, etc.)

### 1.4 Front page language selector component
- **New component**: `components/LanguageSelector.tsx`
  - Position: Top-right of homepage only
  - UI: Simple dropdown button showing current language (EN / FR)
  - Default: `en` selected
  - Persist via `localStorage` key `papope_lang`
  - Use `useI18n()` hook to access `language` and `setLanguage`
  - Only rendered on homepage (`app/page.tsx`), not in app-wide navigation/layout

## Key Design Decisions

1. **No external i18n library**: Build a lightweight custom solution (~200 lines) instead of adding `next-intl` or `react-i18next`. Keeps bundle size minimal and avoids routing complexity.

2. **Flat key structure**: Use dot-notation keys (`common.logout`, `keeppushing.nutrition.addSupplement`) for easy lookup.

3. **Template strings**: Support interpolation with `{variable}` syntax for dynamic content like counts and names.

4. **Language persistence**: Store preference in `localStorage` under `papope_lang`. Default is `'en'`.

5. **AI coach remains French**: The Denis coach character keeps French responses as a personality trait. Only the UI around the chat is translated.

6. **Type-safe translations**: Use TypeScript to ensure all referenced keys exist in the dictionary.
