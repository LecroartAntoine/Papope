# Plan: The Librarian — AI Chat for Chronicle

## Context & Findings

### Codebase observations
- **DB**: Uses `@vercel/postgres` (`sql` tagged template) — **not** Drizzle ORM. The plan's `lib/db.ts` + schema import pattern does not apply. All existing DB calls use raw SQL via `@vercel/postgres`.
- **Auth**: `lib/authOptions.ts` (not `lib/auth.ts` as stated in the plan). Import must be `@/lib/authOptions`.
- **Gemini**: `GEMINI_API_KEY` already exists in `.env.local`. The oracle route calls the Gemini REST API directly (no SDK). The plan proposes `@google/generative-ai` SDK, but to stay consistent with the codebase pattern we should use the REST API directly — **no new dependency needed**.
- **i18n**: All pages use `useI18n()` / `t()` from `@/lib/i18n/context`. Translations live in `lib/i18n/translations.ts` for both `en` and `fr`.
- **Access control**: Chronicle has a layout (`app/chronicle/layout.tsx`) that calls `requireAccess('chronicle')` — the new `/chronicle/librarian` route inherits this protection automatically. No separate session check needed in the page.
- **CSS**: Chronicle uses CSS Modules (`chronicle.module.css`) with a dark fantasy aesthetic. The Librarian page will use its own module CSS to match the theme.
- **No `@google/generative-ai` package**: The oracle route uses the Gemini REST endpoint directly. We'll mirror that pattern — zero new npm installs needed.
- **Gemini model**: The plan mentions `gemini-2.5-flash`. The oracle uses `gemma-4-31b-it`. We'll use `gemini-2.5-flash` for the librarian (it supports `systemInstruction` + multi-turn chat history via the REST API's `contents` array format).
- **Chronicle layout already protects**: No need for `redirect` in the page itself beyond relying on the layout.

## Step-by-Step Implementation

### Step 1 — `lib/chronicle/librarian-context.ts`

Use `@vercel/postgres` (matching existing code), **not** Drizzle:

```ts
import { sql } from '@vercel/postgres'

export async function buildLibraryContext(): Promise<string> {
  const { rows: books } = await sql`
    SELECT id, title, author, categories, added_by
    FROM chronicle_books
    ORDER BY added_at DESC
  `
  const { rows: reviews } = await sql`
    SELECT book_id, reviewer_name, rating, recommendation, date_read
    FROM chronicle_reviews
  `

  const data = books.map((book) => ({
    ...book,
    reviews: reviews
      .filter((r) => r.book_id === book.id)
      .map((r) => ({
        reader: r.reviewer_name,
        rating: r.rating,
        recommendation: r.recommendation,
        dateRead: r.date_read,
      })),
  }))

  return JSON.stringify(data, null, 2)
}
```

### Step 2 — `app/api/chronicle/librarian/route.ts`

Use the Gemini REST API directly (same pattern as oracle), streaming via `streamGenerateContent`. Include `systemInstruction` and pass the full `contents` array (prior turns + new message).

```ts
import { NextRequest } from 'next/server'
import { getServerSession } from 'next-auth'
import { authOptions } from '@/lib/authOptions'
import { buildLibraryContext } from '@/lib/chronicle/librarian-context'

export async function POST(req: NextRequest) {
  const session = await getServerSession(authOptions)
  if (!session) return new Response('Unauthorized', { status: 401 })

  const currentUser = session.user?.name ?? 'Unknown reader'
  const { messages } = await req.json()
  // messages: Array<{ role: "user" | "model"; parts: [{ text: string }] }>

  const libraryData = await buildLibraryContext()

  const systemPrompt = `You are The Librarian (Le Bibliothécaire), a warm and knowledgeable assistant for a private reading club.
The user currently talking to you is: ${currentUser}.

You have full access to the club's reading database below. Use it to answer questions about:
- What books the club has read
- Individual or group ratings and recommendations
- What books a specific person has read or rated highly
- Books two people might enjoy reading together (find overlap in their tastes)
- Recommending what to read next based on someone's history
- Comparing reading habits between members

Always refer to members by their first name. Be warm, specific, and reference actual data from the database when answering. If a question has no relevant data, say so honestly.
Respond in the same language the user writes in (French or English).

--- LIBRARY DATABASE (JSON) ---
${libraryData}
--- END OF DATABASE ---`

  const body = JSON.stringify({
    system_instruction: { parts: [{ text: systemPrompt }] },
    contents: messages, // pass full history including last message
    generationConfig: { temperature: 0.7 },
  })

  const geminiRes = await fetch(
    `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:streamGenerateContent?alt=sse&key=${process.env.GEMINI_API_KEY}`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body,
    }
  )

  if (!geminiRes.ok) {
    return new Response(`Gemini error: ${await geminiRes.text()}`, { status: 502 })
  }

  // Pipe the SSE stream from Gemini → client as plain text tokens
  const stream = new ReadableStream({
    async start(controller) {
      const reader = geminiRes.body!.getReader()
      const decoder = new TextDecoder()
      while (true) {
        const { done, value } = await reader.read()
        if (done) break
        const chunk = decoder.decode(value)
        // SSE lines look like: data: {"candidates":[{"content":{"parts":[{"text":"..."}]}}]}
        for (const line of chunk.split('\n')) {
          if (!line.startsWith('data: ')) continue
          const json = line.slice(6).trim()
          if (!json || json === '[DONE]') continue
          try {
            const parsed = JSON.parse(json)
            const text = parsed?.candidates?.[0]?.content?.parts?.[0]?.text
            if (text) controller.enqueue(new TextEncoder().encode(text))
          } catch {
            // skip malformed SSE lines
          }
        }
      }
      controller.close()
    },
  })

  return new Response(stream, {
    headers: { 'Content-Type': 'text/plain; charset=utf-8' },
  })
}
```

## Key Decisions / Clarifications Needed

1. **No new npm package**: Using the Gemini REST API directly (same as oracle) — no `@google/generative-ai` install.
2. **Streaming via SSE**: `streamGenerateContent?alt=sse` gives server-sent events; we parse `data:` lines and forward the text chunks.
3. **Access control**: Chronicle layout already calls `requireAccess('chronicle')` — the librarian sub-route inherits this. No extra session guard in the page.
4. **i18n language detection**: The system prompt instructs Gemini to reply in the language the user writes in, which naturally handles FR/EN without detecting locale server-side.
5. **No nav bar change needed**: The Chronicle page itself gets a button/link, which is simpler than modifying a separate nav component (there is no standalone nav component for Chronicle — it has its own `back-nav` link pattern).

## What is NOT in scope
- No new DB tables or migrations
- No persistent conversation history (resets on page reload)
- No vector search (acceptable at small scale)
- No changes to homepage cards (the Librarian is accessed from within /chronicle)
